## 크몽 시장 분석 핵심 결과

### 시장 규모
- 홈페이지 제작 카테고리만 13,700+ 서비스 등록
- 대부분 PHP/그누보드/WordPress/아임웹 기반 (90%+)
- **React/Next.js 전문 서비스는 극소수 → 블루오션**

### 경쟁자 분석 (계층별)
- **Tier 1 (200+ 리뷰, 수상):** 더블유포지션(280+), 웹드림(940+), 모두메이드(223) — 주로 에이전시, PHP 기반
- **Tier 2 (30~200 리뷰):** NERD(165, WP), 인생개발자(116, 비즈니스), 파인어플(62, 프리미엄)
- **Tier 3 (0~30 리뷰, React):** 코딩하는곰, 우아한웹, 개발자강팀장 — 대부분 실패 (가격 5만원 등 전략 미스)
- **Prime (상위 2%):** "디자인부터 퍼블리싱까지" — React/Next.js/TS/Docker/AWS, 프로젝트 단가 방식. **나와 거의 동일한 스택으로 Prime 달성한 사례**

### 핵심 발견
- React 서비스가 실패한 이유: 너무 낮은 가격(5만원), 포트폴리오 부재, 차별화 메시지 없음
- 성공한 셀러들의 공통 패턴: 빠른 응답, 친절한 소통, 기대 초과 결과물, 포트폴리오 스크린샷 충실

## 전략: Web-First 3단계

### Phase 1 (1~4주): 리뷰 5개 확보
- 서비스 2개 등록 (랜딩페이지 25만원, 웹사이트 80만원)
- 저가 전략으로 빠른 거래 + 리뷰 축적
- 매 프로젝트 기대 이상 납품 → 5점 리뷰 유도

### Phase 2 (1~3개월): 15+ 리뷰, 가격 정상화
- 서비스 5개로 확장 (풀스택 추가)
- 가격: 랜딩 30~80만, 홈페이지 80~200만, 풀스택 100~300만
- "빠른 응답" 뱃지 획득 목표

### Phase 3 (3~6개월): 월 400만+ 매출
- 프리미엄 포지셔닝
- 앱 서비스 추가 (스토어 관련 면책조항 포함)
- 반복 의뢰 + 유지보수 계약

## 포트폴리오 프로젝트 3개 (현재 1번 진행 중)

### 프로젝트 1: "FlowDesk" SaaS 랜딩페이지 ← 지금 이거
- 1주차 작업
- 가상의 프로젝트 관리 툴
- Next.js + Tailwind + shadcn/ui + Framer Motion
- 배포: Vercel
- → 크몽 서비스 "React 랜딩페이지"의 증거물

### 프로젝트 2: "NexaBridge" 기업 홈페이지 (5P)
- 2주차 작업
- 가상의 IT 컨설팅 회사
- 5페이지: Home / About / Services / Portfolio / Contact
- 카카오맵 연동, 문의 폼 (React Hook Form)
- → 크몽 서비스 "React 기업 홈페이지"의 증거물

### 프로젝트 3: "OrderFlow" 풀스택 대시보드
- 3~4주차 작업
- 주문 관리 대시보드
- 로그인 + 대시보드(차트) + 주문 CRUD + 테이블
- 프론트: Next.js + shadcn/ui + Recharts + TanStack Table
- 백엔드: Node.js + Express + MySQL (or Supabase)
- → 크몽 서비스 "풀스택 웹서비스"의 증거물

## 차별화 전략 요약

**내가 강조해야 할 것:**
- "React 전문" — 크몽에서 React 서비스는 극소수
- "프론트+백엔드+DB+배포 = 1명이 전부" — 커뮤니케이션 오버헤드 제로
- SPA 기반이라 로딩 빠르고 전환 부드럽다는 점을 데모로 증명
- 타겟: 스타트업, IT 기업, 모던 브랜드 (전통 중소기업 X)

**절대 하면 안 되는 것:**
- 5만원 같은 헐값 서비스 등록
- "홈페이지 만듭니다" 같은 범용 포지셔닝
- WordPress/아임웹 카테고리에서 경쟁
- 앱 서비스를 리뷰 없이 먼저 등록
